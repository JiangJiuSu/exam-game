# 📚 考研游戏系统

> 把考研学习做成游戏，通过倒计时任务、连续签到倍率、目标学校进度条等机制，培养按时学习的习惯。

---

## 🎮 核心功能

### ⏱️ 学习时长系统

| 时长类型 | 说明 |
|---------|------|
| 预定时长 | 创建任务时设定，用于倒计时 |
| 真实时长 | 实际学习时间（管理员可修改） |
| 有效时长 | 最终计入总时长的值（自动计算） |

**计算规则：**
- 按时完成 → 有效时长 = 真实时长 × 倍率
- 超时完成 → 有效时长 = 真实时长 × 70% × 倍率

### 🔥 连续签到倍率

| 连续天数 | 倍率 |
|---------|------|
| 第1天 | ×1.1 |
| 第2天 | ×1.2 |
| 第3天 | ×1.3 |
| 第4天 | ×1.4 |
| 第5天+ | ×1.5（封顶） |

断签1天 → 倍率归零，从×1.1重新开始。

### 🎯 目标学校进度条

| 梯度 | 所需学习时长 |
|------|------------|
| 普通院校 | 0小时 |
| 211 | 5,000小时 |
| 985 | 15,000小时 |
| 顶尖985 | 30,000小时 |

支持自定义目标，可随时切换。

### 📝 任务管理

- **倒计时模式**：设定时长，点击"开始任务"后倒计时，实时显示剩余时间
- **截止日期模式**：设定截止时间
- **逾期处理**：倒计时到期自动标记，补完扣30%
- **日历回看**：按日期查看历史完成/逾期任务

### 🏆 成就徽章（24个）

- 学科里程碑（每科4个：10h/50h/100h/500h）
- 签到里程碑（7天/30天/100天）
- 总学习时长里程碑（100h/500h/1000h/3000h）

### 🔧 管理员模式

- 撤销已完成任务（回退学习时长）
- 编辑真实学习时长（自动重算有效时长）
- 删除任务自动回退时长

## 💾 数据安全

| 机制 | 说明 |
|------|------|
| 自动保存 | 每5秒写入本地 |
| 自动备份 | 保留最近10份历史备份 |
| 版本迁移 | 数据结构升级时自动转换 |
| 导入/导出 | JSON 文件备份/恢复 |

数据存储位置：`C:\Users\<用户名>\AppData\Roaming\exam-game\config.json`

## 🚀 快速开始

### 环境要求

- Node.js 18+
- npm

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/JiangJiuSu/exam-game.git
cd exam-game

# 安装依赖
npm install

# 启动网页版
npm run dev

# 启动桌面版
npm run electron:dev
```

### 桌面启动器

项目包含 `启动考研游戏.vbs`，双击即可启动桌面版（无黑窗口）。

## 📁 项目结构

```

├── electron/

│   └── preload.js             # IPC 桥接

│   ├── utils/
│   │   ├── achievements.js    # 成就定义
│   │   ├── dataPersistence.js # 数据持久化/备份
│   │   └── initApp.js         # 应用初始化
│   ├── components/            # UI 组件
│   └── views/                 # 页面
├── DESIGN.md                  # 游戏机制设计文档
├── PLAN.md                    # 技术实现计划
└── README.md                  # 本文件
```
